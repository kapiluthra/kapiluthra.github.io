Before you start

✓ Familiarity with vector search / RAG basics. Start with Building a Production RAG Pipeline on OpenText if you're new to this stack.
✓ Basic understanding of OpenText Content Server (node IDs, workspaces, OTCSTicket auth). The articles explain OTCS-specific concepts as they arise.
✓ Python comfort level — code examples use Python 3.10+. The patterns translate to any language.
01
AI/RAG · Security
ACL-Aware RAG: Why Your Enterprise AI Must Respect Document Permissions
Naive RAG leaks confidential documents across users — it's the #1 reason enterprise AI projects get killed. This article covers the only safe architecture: pre-retrieval filtering using a two-tier permission cache keyed to OTCSTickets. Includes a complete PermissionResolver implementation and test methodology.

02
AI/RAG · Architecture
GraphRAG on OpenText: Building a Knowledge Graph from Your Content Repository
Flat vector search can't answer multi-hop questions like "find all contracts referencing supplier X that were approved by Legal." GraphRAG maps your ECM structure — containment, authorship, approvals, categories — as graph edges, then routes queries to graph traversal or vector search depending on their shape.

AI/RAG · Agents
Agentic ECM: When Your AI Decides Which Documents to Fetch
When a query requires fetching a contract, reading an amendment, checking workflow history, and cross-referencing a policy — you need an agent, not a search engine. This article covers the tool surface design, agent loop with MAX_HOPS guardrails, three production patterns, and the hard parts: tool latency, hallucinated node IDs, and infinite reasoning loops.
